# Starter Pack per Nuovo Repo

Questo pacchetto contiene versioni riusabili e neutrali dei documenti di pianificazione per il nuovo progetto sito.

## Contenuto
- `PLAN_TEMPLATE.md`: piano strategico completo.
- `CHECKLIST_TEMPLATE.md`: checklist operativa da spuntare.
- `TIMELINE_12_WEEKS_TEMPLATE.md`: roadmap settimanale esecutiva.

## Come usarlo nel nuovo repo
1. Copia la cartella `NEW_REPO_STARTER` nel nuovo repository.
2. Rinomina i file eliminando `_TEMPLATE`.
3. Sostituisci i placeholder `[DA DEFINIRE]`.
4. Esegui un primo commit dedicato alla pianificazione.

## Placeholder da completare subito
- Nome progetto
- Obiettivo 90 giorni
- Ambito geografico
- Budget mensile
- Offerta principale (servizi)
- CTA primaria

## Nota operativa
Se vuoi mantenere separati strategia e produzione:
- aggiorna `PLAN_TEMPLATE.md` quando cambi direzione;
- aggiorna `CHECKLIST_TEMPLATE.md` durante l'esecuzione;
- usa `TIMELINE_12_WEEKS_TEMPLATE.md` come planning settimanale reale.
